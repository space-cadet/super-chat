## Details

Implement the manual multi-step tool calling loop that enables LLM-native tool execution. This is the heart of the system — extracted and refined from obsidian-ai's proven implementation.

## Architecture

```
AgentLoop.run(session, messages, tools, signal)
├── Step 1: LLM stream with tools (single step)
│   ├── Yields text-delta events → onTextDelta callback
│   ├── Collects tool-call events → pendingCalls[]
│   └── Detects finish → check if tools pending
├── If tools pending:
│   ├── Approval check (autoApply or requestApproval callback)
│   ├── Execute approved tools via ToolExecutor
│   ├── Yield tool-result / tool-error → onToolResult callback
│   └── Rebuild messages with tool context → loop to Step 1
└── If no tools or maxSteps reached → return final text
```

## Key Design Decisions

1. **Callback-based API** (not pure async generator) — approval requires UI interaction mid-stream
2. **Manual loop** (not SDK ToolLoopAgent) — full control over approval flow and message reconstruction
3. **Tool result formatting** — pluggable `ToolResultFormatter` prevents raw JSON dumps in LLM responses
4. **Message reconstruction** — assistant message (text + tool calls) + tool result messages fed back to LLM

## API

```typescript
class AgentLoop {
  constructor(
    opts: AgentLoopOptions,
    formatResult?: ToolResultFormatter
  )

  async run(
    session: ChatSession,
    messages: ChatMessage[],
    tools: ToolDefinition[],
    signal?: AbortSignal
  ): Promise<AgentLoopResult>
}
```

## Callbacks

| Callback | Purpose | Required |
|----------|---------|----------|
| `onTextDelta` | Stream text chunks to UI | No |
| `onToolCall` | Notify UI of pending tool call | No |
| `onToolResult` | Notify UI of tool result | No |
| `requestApproval` | Pause for user approval | No (autoApprove=true) |
