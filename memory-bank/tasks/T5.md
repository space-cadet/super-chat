## Details

Define the adapter interfaces that allow the core to work with any LLM provider, persistence layer, RAG system, or context source. These are the extension points.

## Adapter Interfaces

### LLMAdapter
```typescript
interface LLMAdapter {
  // Simple text streaming
  streamChat(
    messages: { role: string; content: string }[],
    signal?: AbortSignal
  ): AsyncIterable<string>;

  // Tool-capable streaming
  streamChatWithTools(
    messages: { role: string; content: string }[],
    tools: ToolDefinition[],
    signal?: AbortSignal
  ): AsyncIterable<StreamEvent>;

  // Provider info
  getProviders(): ProviderInfo[];
  getModels(provider: string): ModelInfo[];
  testConnection(): Promise<{ ok: boolean; message: string }>;
}
```

### PersistenceAdapter
```typescript
interface PersistenceAdapter {
  loadSessions(): Promise<ChatSession[]>;
  saveSession(session: ChatSession): Promise<void>;
  deleteSession(sessionId: string): Promise<void>;
  archiveSession(sessionId: string): Promise<void>;
}
```

### RAGAdapter
```typescript
interface RAGAdapter {
  analyzeQuery(query: string): Promise<QueryAnalysisResult>;
  retrievePapers(analysis: QueryAnalysisResult): Promise<RetrievedPaper[]>;
  buildContext(papers: RetrievedPaper[]): Promise<string>;
}
```

### ContextAdapter
```typescript
interface ContextAdapter {
  searchMentions(query: string): Promise<ContextItem[]>;
  getActiveDocument(): Promise<ContextItem | null>;
  getSelection(): Promise<ContextItem | null>;
  resolveEmbed(link: string): Promise<string>;
}
```

## Reference Implementations

- `VercelLLMAdapter` — wraps Vercel AI SDK (9 providers)
- `LocalStoragePersistence` — browser localStorage
- `MemoryPersistence` — in-memory (for testing)
- `DemoToolAdapter` — mock tools (calculate, weather, arxiv)

## Design Decisions
- Interfaces are minimal — only what's needed by the core
- Reference implementations provided for common cases
- Consumers can implement custom adapters for any provider
- AbortSignal support for cancellation throughout
- **Vercel AI SDK v6 integration**: `streamText` with `stopWhen: stepCountIs(1)` for single-step streaming — AgentLoop handles multi-step
- **Lazy provider loading**: `import()` inside factory functions to avoid bundling unused providers
- **JSON Schema → Zod**: Adapter converts `ToolDefinition.parameters` to Zod schemas for SDK compatibility
- **pnpm over npm**: Switched to pnpm for reliable dependency resolution (npm was failing to install devDependencies)

## Lessons Learned
- Vercel AI SDK v6 uses `UIMessage` with `id`, `role`, `parts` — not simple `{role, content}` objects. Need `convertToModelMessages()`.
- `streamText` with tools needs explicit `stopWhen` or it defaults to `stepCountIs(20)`. We use `stepCountIs(1)` for single-step to let AgentLoop control multi-step.
- SDK chunk types: `text-delta` has `text` property (not `textDelta` in v6), `tool-call` is `TypedToolCall` shape, `finish-step` replaces `step-finish`.
- npm was failing to install scoped packages in devDependencies — pnpm resolved it immediately.
- Provider packages should be devDependencies (not bundled into library) — consumers install only the providers they use.
