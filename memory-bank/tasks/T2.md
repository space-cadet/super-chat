## Details

Define the core type system that insulates consumers from Vercel AI SDK changes. The StreamEvent union is the contract between the LLM adapter layer and the UI layer.

## Types to Define

### Message Types
- `ChatMessage` — id, role, content, timestamp, metadata, toolCalls, toolResults, citations
- `ChatSession` — id, title, messages, createdAt, updatedAt, llmProvider, llmModel
- `MessageRole` — 'user' | 'assistant' | 'system'

### Tool Types
- `ToolCall` — id, name, args (Record<string, unknown>)
- `ToolResult` — success?, content?, error?, path?
- `ToolDefinition` — name, description, parameters (JSON Schema)
- `ToolAdapter` — interface: getAvailableTools(), executeTool(call)

### Stream Events (SDK-agnostic)
- `text-delta` — streaming text chunk
- `tool-call` — LLM requested a tool
- `tool-result` — tool executed successfully
- `tool-error` — tool execution failed
- `citation` — RAG papers retrieved
- `rag-status` — retrieval progress updates
- `step-finish` — one tool step completed
- `finish` — stream complete
- `error` — fatal error

### Adapter Interfaces
- `LLMAdapter` — streamChat(), streamChatWithTools(), getProviders(), getModels(), testConnection()
- `PersistenceAdapter` — loadSessions(), saveSession(), deleteSession(), archiveSession()
- `RAGAdapter` — analyzeQuery(), retrievePapers(), buildContext()
- `ContextAdapter` — searchMentions(), getActiveDocument(), getSelection(), resolveEmbed()

### Settings Types
- `ProviderProfile` — id, name, provider, model, apiKey, baseUrl, etc.
- `ProviderType` — union of supported providers
- `ChatSettings` — enableRAG, enableTools, maxAgentSteps, autoApply, etc.

## Design Decisions
- StreamEvent is a discriminated union — consumers pattern-match on `type`
- ToolDefinition uses JSON Schema (not Zod) — framework-agnostic
- All IDs are strings (UUIDs)
- Timestamps are numbers (Unix ms)
