# T20: Fix ChatEngine Real-Time Streaming for Tools

*Created: 2026-06-20 01:00:00 IST*
*Related: Finding 1 from Comprehensive Survey*

**Status**: ✅ **COMPLETED** 2026-06-20
**Priority**: CRITICAL
**Phase**: Foundation Fix

## Problem Statement
`ChatEngine.runWithTools()` in `super-chat/src/core/ChatEngine.ts` did NOT stream in real-time when tools were enabled. The method waited for `AgentLoop.run()` to complete entirely, then yielded the final text as a single chunk.

## Solution
Changed `AgentLoop.run()` from `Promise<AgentLoopResult>` to `AsyncGenerator<StreamEvent, AgentLoopResult>`.

### New AgentLoop Interface
```typescript
// BEFORE:
async run(session, messages, tools, signal): Promise<AgentLoopResult>

// AFTER:
async *run(session, messages, tools, signal): AsyncGenerator<StreamEvent, AgentLoopResult>
```

### Event Flow
```
Step 0: User asks question
  → "text-delta": "I'll search for..."
  → "text-delta": "notes about..."
  → "tool-call": { name: "search_notes", args: {...} }
  → "tool-result": { call: {...}, result: {...} }
Step 1: LLM processes result
  → "text-delta": "Based on the search..."
  → "text-delta": "I found..."
[No more tool calls]
  → "finish": { reason: "complete" }
  → return AgentLoopResult
```
