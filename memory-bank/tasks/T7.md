## Details

Parse `@AgentName` mentions in user messages to route queries to specific agents. Also supports `@` for context mentions (documents, notes, etc.).

## Features

### Agent Mentions
- `@Cloudy` — route to Cloudy agent
- `@Gemini @Cloudy` — route to multiple agents
- `@all` — route to all agents (explicit)
- No mention — default behavior (all agents or active agent)

### Context Mentions
- `@DocumentName` — inject document into context
- `@NoteTitle` — inject note content
- Fuzzy search for matching names
- Deduplication (same mention multiple times)

## API

```typescript
interface Mention {
  type: 'agent' | 'context';
  name: string;
  raw: string;
}

class MentionParser {
  parse(text: string): { mentions: Mention[], cleanText: string };
  hasMentions(text: string): boolean;
  extractAgentNames(text: string): string[];
  extractContextNames(text: string): string[];
}

class MentionResolver {
  constructor(
    agentRegistry: Map<string, AgentEngine>,
    contextAdapter: ContextAdapter
  );
  
  resolveAgents(mentions: Mention[]): AgentEngine[];
  resolveContext(mentions: Mention[]): Promise<ContextItem[]>;
}
```

## Design Decisions
- Regex-based parsing: `@([A-Za-z0-9_]+)`
- Case-insensitive agent matching
- Context mentions resolved via ContextAdapter
- Clean text (mentions stripped) sent to LLM
